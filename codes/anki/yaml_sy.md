
# yaml

## YAML 中注释如何书写？

<!-- notecardId: 1762057043312 -->

以 `#` 开头，直到行尾。

## YAML 的根对象通常是什么数据结构？

<!-- notecardId: 1762057043338 -->

映射（map），等价于字典/哈希/对象。

## YAML 中键值对的基本写法是什么？

<!-- notecardId: 1762057043361 -->

`key: value`（冒号后建议空格）。

## YAML 中布尔和数字如何区分？例如 `1` 与 `true`

<!-- notecardId: 1762057043382 -->

`1` 解析为数值；布尔需写 `true/false`。

## YAML 中的 null 值写法有哪些？

<!-- notecardId: 1762057043400 -->

`null`、`~`、或空值（视解析器）。

## 字符串在 YAML 中必须加引号吗？

<!-- notecardId: 1762057043419 -->

不必须；需要包含特殊字符时可加单/双引号。

## 双引号与单引号在 YAML 中的主要区别？

<!-- notecardId: 1762057043440 -->

双引号支持转义序列；单引号只用 `''` 表示字面量 `'`。

## 在双引号字符串中写换行或控制字符的方式？

<!-- notecardId: 1762057043462 -->

使用转义：如 `\n`、`\t`、`\u263A`、`\x0d\x0a` 等。

## 如何在 YAML 中写 Unicode 码点？

<!-- notecardId: 1762057043479 -->

使用 `\uXXXX`（例如 `\u00B2`）。

## YAML 的字面量块（literal block）使用哪个标记？

<!-- notecardId: 1762057043500 -->

使用 `|`，保留换行。

## YAML 的折叠块（folded block）使用哪个标记？

<!-- notecardId: 1762057043520 -->

使用 `>`，将换行折叠为空格（空行保留为换行）。

## 在多行块中，额外缩进的行如何处理？

<!-- notecardId: 1762057043541 -->

“更缩进”的行保留其换行与缩进。

## YAML 推荐用多少空格缩进？

<!-- notecardId: 1762057043561 -->

通常 2 空格（非必须，一致即可）。

## YAML 映射的键必须是字符串吗？

<!-- notecardId: 1762057043581 -->

不是；可为数值、复合结构甚至序列。

## 如何开始一个复合键（多行键）的书写？

<!-- notecardId: 1762057043601 -->

以 `?` 后接空格，然后给出键值（可多行）。

## YAML 中如何表示“序列”（列表）？

<!-- notecardId: 1762057043621 -->

使用 `-` 开头的行；或 JSON 行内 `[ ... ]`。

## YAML 中如何写“映射（对象）”的行内形式？

<!-- notecardId: 1762057043641 -->

使用 `{key: value, ...}`（引号可选，视内容而定）。

## YAML 与 JSON 的关系是什么？

<!-- notecardId: 1762057043659 -->

YAML 是 JSON 的超集；JSON 文法在 YAML 中有效。

## 如何在 YAML 中复用值？

<!-- notecardId: 1762057043681 -->

用锚 `&name` 定义，用别名 `*name` 引用。

## 如何将一个映射的所有键值“继承/合并”到当前映射？

<!-- notecardId: 1762057043700 -->

使用合并键 `<<: *anchor`。

## `<<` 合并键可以合并多个映射吗？

<!-- notecardId: 1762057043721 -->

可以，使用序列：`<<: [*a, *b]`（解析器需支持）。

## 如何显式声明一个值的类型为字符串（即便看起来像数字）？

<!-- notecardId: 1762057043741 -->

使用标签：`!!str 0.5`。

## 什么是“语言特定标签”的示例？

<!-- notecardId: 1762057043760 -->

`!!python/complex 1+2j`（解析器特定）。

## 可以在复合键上使用标签吗？

<!-- notecardId: 1762057043780 -->

可以，例如 `? !!python/tuple [5, 7] : Fifty Seven`。

## YAML 如何表示 ISO 日期时间？

<!-- notecardId: 1762057043800 -->

如 `2001-12-15T02:59:43.1Z`（可被解析为日期/时间）。

## 如何在 YAML 中嵌入二进制数据？

<!-- notecardId: 1762057043817 -->

使用 `!!binary |` 加 base64 内容。

## YAML 的集合（set）有何两种表示法？

<!-- notecardId: 1762057043838 -->

块状：

```
set:
  ? item1
  ? item2
```

行内：`{item1, item2}`。

## YAML 的集合与“值为 null 的映射”有何关系？

<!-- notecardId: 1762057043858 -->

集合等价于“键存在且值为 null 的映射”。

## YAML 文档如何显式结束？

<!-- notecardId: 1762057043880 -->

使用 `...`。

## 在 YAML 中，`'have ''one'' escape pattern'` 中的两个单引号表示什么？

<!-- notecardId: 1762057043901 -->

表示字面量的单引号字符 `'`。

## 在 YAML 中，`"have many: \", \0, \t"` 里的 `\"` 表示什么？

<!-- notecardId: 1762057043919 -->

表示一个双引号字符。

## 行内映射 `{key: [3, 2, 1, takeoff]}` 中 `takeoff` 省略引号意味着什么？

<!-- notecardId: 1762057043939 -->

一个不含特殊字符的裸字符串标量。

## 下面的键定义是否有效：`0.25: a float key`？

<!-- notecardId: 1762057043961 -->

有效；数字可作键。

## 复合键配合序列作键的示例？

<!-- notecardId: 1762057043980 -->

```
? - Manchester United
  - Real Madrid
: [2001-01-01, 2002-02-02]
```

## 在 YAML 中嵌套序列的一个简洁写法？

<!-- notecardId: 1762057043999 -->

`- - - Nested`（多层 `-` 可合并为行内结构）。

## 当使用 `|` 字面量块时，原文中的空行会如何处理？

<!-- notecardId: 1762057044020 -->

保留为空行（换行）。

## 当使用 `>` 折叠块时，普通换行会如何处理？

<!-- notecardId: 1762057044041 -->

折叠为单个空格；空白行仍作为换行。

## YAML 的键中可以包含冒号 `:` 吗？

<!-- notecardId: 1762057044061 -->

可以，但常需加引号，或使用复合键语法。
