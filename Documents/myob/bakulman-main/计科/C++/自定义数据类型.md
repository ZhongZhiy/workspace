1. struct name { .....} **;** 必须跟个分号，否则紧跟一个变量名视为声明或定义
2. 类的数据成员定义和普通类型定义一样，记得[[初始化]]
#### [[构造函数]]
1. 构造函数名和类名相同，没有返回类型，有参数列表（可以为空）和函数体（可以为空），类可以有多个构造函数，但需满足[[重载函数]]要求
2. 用来对类的数据成员初始化
3. 构造函数可以在const对象的构造过程中写值

#### 默认构造函数
类通过一个特殊的构造函数来控制默认初始化过程，这个函数就是默认构造函数，默认构造函数无需实参

没有显式的默认构造函数时，编译器会创建一个合成的默认构造函数（synthesized default constructor）
`classname() = default;`
这是要求编译器实施合成的默认构造函数。我们需要构造其他构造函数就需要它

#### 构造函数初始值列表
- **构造函数初始化列表的语法**：
```cpp
//值也可以用{}
类名(参数列表) : 成员1(值1), 成员2(值2), ... {     // 构造函数体 }

```
当某个类成员被构造函数初始值列表忽视时（没在列表里的），它将以与合成默认构造函数相同的方式隐式地初始化，通常使用类内初值就挺好
```cpp
//只给bookNo赋值了，其他都是默认初始化
sales_date(const string &s) : bookNo(s),units_sold(0),revenue(0){}
```
注意函数体是空的，因为我们只用给数据成员赋初值，没有其他操作

- **优点**：
    
    - 必须用初始化列表初始化 **常量成员** 和 **引用成员**。
    - 必须用初始化列表为类成员对象调用其构造函数。
    - 提高效率，避免多余的默认构造和赋值操作。
- **适用场景**：
    
    - 常量成员、引用成员。
    - 类的成员是其他类的对象。
    - 需要高效初始化时。
