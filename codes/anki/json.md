# JSON
## JSON 是什么？

<!-- notecardId: 1762583321014 -->

JSON 是一种极其简单的数据交换格式。

## JSON 的设计目标是什么？

<!-- notecardId: 1762583321022 -->

对人类 **易读易写**，对机器 **易解析易生成**。

## JSON 的全称是什么？

<!-- notecardId: 1762583321029 -->

JavaScript Object Notation（JavaScript 对象表示法）

## 按 json.org 的定义，JSON 具有哪两个核心特性？

<!-- notecardId: 1762583321035 -->

人类可读、机器可解析。

---


## 一段 JSON 可以是哪些类型的值？

<!-- notecardId: 1762583321040 -->

可以是任意 JSON 值：对象、数组、字符串、数字、布尔值或 null。

## JSON 通常以哪两种形式呈现？

<!-- notecardId: 1762583321045 -->

1. 键值对集合 `{}`
2. 有序的值列表 `[]`

## {{c1::花括号 { }}} 表示{{c2::键值对集合（对象/字典/结构体等）}}。

<!-- notecardId: 1762583321050 -->

## {{c1::方括号 [ ]}} 表示{{c2::有序的值列表（数组/列表/序列等）}}。


## JSON 中对象 `{}` 在不同语言中可能被理解为什么？

<!-- notecardId: 1762583321060 -->

对象 / 记录 / 结构体 / 字典 / 哈希表 / 关联数组

## JSON 中数组 `[]` 在不同语言中可能被理解为什么？

<!-- notecardId: 1762583321069 -->

数组 / 向量 / 列表 / 序列


## 纯正的 JSON 是否允许注释？

<!-- notecardId: 1762583321075 -->

不允许。

## 哪些解析器允许 C 风格注释？

<!-- notecardId: 1762583321080 -->

多数 JSON 解析器。

## C 风格注释包括哪些形式？

<!-- notecardId: 1762583321086 -->

`//` 单行注释 和 `/* ... */` 多行注释。

## JSON 中是否允许 trailing comma（尾逗号）？

<!-- notecardId: 1762583321091 -->

部分解析器允许，但应避免以保证兼容性。


非法（标准 JSON 不允许）：

```json
{
  "a": 1,
}
```



## JSON 支持哪些数据类型？

<!-- notecardId: 1762583321097 -->

字符串、数字、对象、数组、布尔值、null

## JSON 字符串的格式要求是什么？

<!-- notecardId: 1762583321104 -->

字符串必须用 **双引号 `"`** 包裹。

## JSON 数字允许哪些形式？

<!-- notecardId: 1762583321109 -->

整数、小数、科学计数法（例如 `12e10`、`3.141e-10`、`1.23e+4`）

## JSON 对象的语法是什么？

<!-- notecardId: 1762583321115 -->

用花括号包裹的键值对集合，例如：

```json
{ "key": "value" }
```

## JSON 数组的语法是什么？

<!-- notecardId: 1762583321120 -->

用方括号包裹的有序值列表，例如：

```json
["value1", "value2"]
```

## JSON 布尔值如何表示？

<!-- notecardId: 1762583321126 -->

`true` 和 `false`

## JSON 空值如何表示？

<!-- notecardId: 1762583321133 -->

`null`

---


## JSON 中的键必须使用什么形式？

<!-- notecardId: 1762583321138 -->

必须使用 **双引号括起来的字符串**。

## JSON 是否允许单引号键？

<!-- notecardId: 1762583321146 -->

不允许。

## {{c1::键和值之间}} 用 {{c2::冒号 ":"}} 分隔。

<!-- notecardId: 1762583321151 -->

## {{c1::多个键值对}} 之间用 {{c2::逗号 ","}} 分隔。

<!-- notecardId: 1762583321159 -->

## JSON 对象的典型结构？

<!-- notecardId: 1762583321166 -->

```json
{
  "key": "value",
  "number": 42,
  "nested": { "a": 1 },
  "array": [1, 2, 3],
  "bool": true,
  "nothing": null
}
```

## JSON 数组可以包含哪些类型的元素？

<!-- notecardId: 1762583321171 -->

任意类型，包括数字、字符串、对象、数组、布尔、null。

## JSON 是否支持嵌套对象与数组？

<!-- notecardId: 1762583321178 -->

支持任意层级嵌套。

## JSON 对空白字符的处理规则是什么？

<!-- notecardId: 1762583321182 -->

空白字符（空格、换行、缩进）**不影响解析**，可自由使用以增强可读性。



## 为保证兼容性，JSON 应避免哪些写法？

<!-- notecardId: 1762583321187 -->

1. C 风格注释
2. 尾逗号
3. 单引号字符串
4. 未加引号的键名



## JSON 的核心设计哲学是什么？

<!-- notecardId: 1762583321192 -->

**简单 + 通用 + 可移植**

## JSON 的文件后缀名是什么？

<!-- notecardId: 1762583321197 -->

`.json`

## JSON 的 MIME 类型是什么？

<!-- notecardId: 1762583321201 -->

`application/json`
